Tokens frequency

Por Guilherme Dal Bianco, UFFS Brazil

Timelimit: 1

A hospital aims at migrating the patient system to a new one. However, it is observed that the client table has a problem with the duplicate register. The same patient can be stored in the database in two different ways, i.e., “ketlyn da Silva” or “ketlin da Silva”. To mitigate it, the number of common tokens can be computed by splitting the string into segments (tokens) with defined size. For example, the tokens with size three for the string “ketlyn” are represented as: “ket”,”etl”, ”tly” e “lyn”; already the string “ketlin” produces the tokens: “ket”, “etl”, “tli”,”lin”. Observe that the token generation is produced by sliding windows approach, sliding one character at each time. In the end, it is computed the number of common tokens. The task here is to produce and identify the token with size two (2) more frequent in a given string. When to token frequency is equal, the first token in alphabetic order should be outputted. The string must be converted to lower case.

Input

One input line composed of a string S representing the text.

Output

The more frequent token in the string S defining a token size of two (2) followed by the frequency number is printed.

Input Samples Output Samples

casa carro casa

ca:3

casa

as:1

ewrwsffdes

de:1
